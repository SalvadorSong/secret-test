# BBS Balance Music Game

## 项目概述
基于摄像头的BBS（Berg平衡量表）音乐化康复训练游戏系统。患者的平衡动作实时生成音乐反馈，同时自动采集运动学数据用于BBS评分预测。

## 核心理念
**"身体就是乐器"** — 站立、重心转移、转身等平衡动作实时控制音乐生成。

## 技术栈
- **姿态检测**: MediaPipe Pose Landmarker (lite, CDN加载, 33个关键点)
- **音频合成**: Tone.js
- **部署**: GitHub Pages (纯前端, 单HTML文件)
- **备选升级**: RTMPose (MMPose, 133个关键点, 更高精度)

## 5个小游戏设计

### Game 1: 节拍树 (Rhythm Tree) ✅ 已完成
- **BBS对应项**: #1 坐→站, #2 站→坐
- **交互**: 站起→上行音阶(do-re-mi-fa-sol), 坐下→下行音阶+和弦
- **视觉**: 树从🌱→🌿→🌳→🌳🌸逐步成长
- **数据采集**: 坐站次数、转换速度、站立/坐下时间、髋关节角度、膝关节角度、重心稳定性

### Game 2: 平衡DJ (Balance DJ) 🔲 待开发
- **BBS对应项**: #6 闭眼站立, #7 双脚并拢站立
- **交互**: 身体重心X轴→左右声道平衡(pan), Y轴→音量; 越稳定音乐越丰富
- **数据采集**: 重心轨迹(COP), AP/ML晃动幅度, 闭眼vs睁眼稳定性差异

### Game 3: 旋律之路 (Melody Path) 🔲 待开发
- **BBS对应项**: #8 前伸, #10 转头看后方
- **交互**: 前伸距离→音高, 转头角度→立体声位置, 动作流畅度→legato/staccato
- **数据采集**: 最大前伸距离, 转头范围, 重心前移稳定性

### Game 4: 舞步作曲 (Dance Composer) 🔲 待开发
- **BBS对应项**: #11 原地转身360°, #12 交替踏步
- **交互**: 转身角度→音阶位置, 踏步→鼓点(左右脚不同鼓声)
- **数据采集**: 转身时间, 踏步频率和对称性, 踏步高度

### Game 5: 独奏时刻 (Solo Moment) 🔲 待开发
- **BBS对应项**: #13 前后脚站立, #14 单腿站立
- **交互**: 维持时间→音乐长度, 微小晃动→颤音/震音效果
- **数据采集**: 维持时间, 重心偏移轨迹

## 坐/站检测逻辑
- 使用**髋关节角度** (肩膀-髋部-膝盖三点夹角) 判断坐/站状态
- 站立: 髋角 > 160°
- 坐下: 髋角 < 120°
- 过渡: 120°-160° (不触发音乐)
- 不依赖脚踝可见性

## 已知问题 & 待解决
1. 摄像头切换(前置/后置)在部分手机上可能不稳定
2. 角度精度需要验证 (MediaPipe vs 专业设备约±5-10°误差)
3. MediaPipe对运动功能受限的患侧肢体识别率待测试
